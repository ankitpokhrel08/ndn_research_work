# Data

This repository contains **code, notebooks, figures, trained models, scenario
sources, and documentation** (~43 MB). The **raw data and large derived feature
CSVs are not committed** - they are provided as a separate downloadable bundle to
keep the repo lean. The analysis pipeline regenerates every CSV (and all figures)
from the raw data.

---

## What's in the repo vs. the data bundle

| In the repo (committed) | In the data bundle (download) |
|---|---|
| All notebooks + scenario `.cpp` sources | NDNsim raw tracer output (`*-rate-trace.txt`, etc.) |
| All figures (`figures/`) | miniNDN raw NFD logs (`*_metrics.jsonl`) |
| Trained models (`research_analysis/models/`) | NDNsim feature CSVs (`processed/*.csv`) |
| Tiny result tables (`ks_results.csv`, `detection_latency.csv`) | miniNDN feature CSVs (`Datacard/Datasets/*.csv`) |
| All findings docs + README | |

miniNDN/Datacard/Logs/                              ← raw miniNDN logs
miniNDN/Datacard/Datasets/                          ← (regenerated, or from bundle)
```

## Reproduction chain (what regenerates from what)

```
NDNsim:   scenarios/*.cpp ──[ndnSIM]──► results/*.txt ──[01_preproc]──► processed/*.csv ──[02,03]──► figures
miniNDN:  miniNDN emulator ───────────► Datacard/Logs/*.jsonl ──[research_analysis]──► models, figures
```

Two important notes:

1. **The CSVs regenerate from the raw data, not from the `.cpp` scenarios directly.**
   You need the raw-data bundle (or a full ndnSIM/ns-3 build to re-run the
   simulations) to rebuild the NDNsim CSVs.
2. **The miniNDN raw logs cannot be regenerated from this repo** - they are emulator
   captures and are the rawest source. They live only in the data bundle.

So: with the **figures and trained models already in the repo**, the results are
viewable and usable as-is. To **re-run the pipeline end-to-end**, download the bundle
first.

## Reproduce from the bundle

```bash
# 1. Download the bundle and extract into the paths listed above.

# 2. NDNsim (raw traces → CSVs → figures)
cd NDNsim/notebooks
jupyter nbconvert --to notebook --execute --inplace 01_preprocessing.ipynb
jupyter nbconvert --to notebook --execute --inplace 02_eda.ipynb
jupyter nbconvert --to notebook --execute --inplace 03_updated_detection.ipynb

# 3. miniNDN (raw logs → models → figures)
cd ../../miniNDN/research_analysis
jupyter nbconvert --to notebook --execute --inplace 01_eda.ipynb
jupyter nbconvert --to notebook --execute --inplace 02_global_model.ipynb
jupyter nbconvert --to notebook --execute --inplace 03_per_topology_models.ipynb
```

## Maintainer: how to build the bundle

```bash
# from the repo root - zips exactly the git-ignored data
zip -r ndn-data-bundle.zip \
    NDNsim/ndnsim-research-main/ndn-research/results \
    NDNsim/processed \
    miniNDN/Datacard/Logs \
    miniNDN/Datacard/Datasets \
    -x '*.DS_Store'

# attach to a GitHub Release (after the repo is pushed)
gh release create v1.0 ndn-data-bundle.zip \
    --title "v1.0 - data bundle" \
    --notes "Raw traces/logs + derived feature CSVs for the NDN anomaly-detection pipeline."
```

Then paste the release asset URL into the **Download** section above.
